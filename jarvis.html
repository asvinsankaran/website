JARVIS - Personal Project

Github ›

Given my passion for engineering, I enjoy pursuing personal projects in my free time such as JARVIS, a sky-cam-like robot for my room, capable of navigating the space and interacting with users. As a module suspended by 4 winch-controlled ropes, I created detailed CAD designs on OnShape for 3D printing winch blocks, motor adaptors, and pulley joints. I then designed electrical circuits for microcontroller-to-motor communication with an Arduino MEGA, and controls firmware in C++. Furthermore, I am using OpenCV in MATLAB for computer vision, and motor encoders for triangulation (quadrilateration) using a gradient descent algorithm in C++. I also use vector calculus for smooth movement in 3D space, as well as a PID layer for motor PWM control. I hope to soon increase sophistication by implementing RF communication, SIFT object detection, end-effector manipulation, and more.

Revision 3 - Improving Reliability and Attempting 3D Control

I made a lot of changes to the hardware and controls of the robot in this revision:

  • Use encoded motors to improve accuracy and decrease latency from the computer vision algorithm
  • Upgraded from an Arduino to an ELEGOO for more interrupt and PWM pins
  • Spent much time researching and testing different 3D triangulation algorithms as there are 3 spatial dimentions (XYZ) and 4 motors (overconstrained)
  • Implemented a gradient descent algorithm to calculate the most likely position of intersection of the wires/spheres for triangulation (quadrilateration)

Revision 2 - SpiderCam Design and 2D Testing

Inspired by SpiderCams - the cameras used in stadiums to record football games - I realized that this is a much simpler hardware design to implement.

I started with the 2D case using 2 winches, an Arduino UNO, and many 3D printed iterations. I used computer vision for positional encoding, where I used a red, green, and blue LED on the module to make triangulation easier by isolating the RGB peaks. From the location, I could calculate a setpoint, path, PID, and PWM duty cycle.

Revision 1 - Cartesian XYZ Design and Prototyping

I started with a 3D printer inspired cartesian model using ropes, pulleys, and winches to create a deterministically controlled robot. In analyzing this design, I realized that it introduces many issues in electrical integration, error propagation, and unrealistic assumptions like ropes that don't slip, don't stretch, and have minimal backlash.
